Introduction

Kendallâ€™s Tau est un estimateur non paramÃ©trique de la corrÃ©lation entre deux ensembles de variables alÃ©atoires, \(X\) et \(Y\), dÃ©fini comme suit :

\[ \tau(X, Y) = \text{P}\left\{ (X - X^*)(Y - Y^*) > 0 \right\} - \text{P}\left\{ (X - X^*)(Y - Y^*) < 0 \right\} \]

OÃ¹ \((X^*, Y^*)\) est une copie indÃ©pendante de \((X, Y)\).

Lâ€™algorithme traditionnel de calcul de cette statistique est donnÃ© par (Press, Flannery, Teukolsky, Vetterling, 1993):

\[ \tau = \frac{C - D}{\sqrt{(C + D + T_1)(C + D + T_2)}} \]

OÃ¹ : - \(C\) est le nombre de concordances, - \(D\) est le nombre de discordances, - \(T_1\) et \(T_2\) sont les nombres de liaisons dans les deux sÃ©ries.

Les dÃ©finitions des termes sont les suivantes :

  • Concordant (c) : Un couple \((i, j)\) est concordant si \((X_i - X_j)(Y_i - Y_j) > 0\). Cela signifie que les rangs relatifs de \(X\) et \(Y\) sont dans le mÃªme ordre pour les deux observations.

  • Discordant (d) : Un couple \((i, j)\) est discordant si \((X_i - X_j)(Y_i - Y_j) < 0\). Cela signifie que les rangs relatifs de \(X\) et \(Y\) sont dans un ordre opposÃ© pour les deux observations.

  • Liaison T1 (T_1) : Un couple \((i, j)\) est une liaison \(T_1\) si \(X_i = X_j\) et \(Y_i \neq Y_j\). Cela signifie que les valeurs de \(X\) sont Ã©gales, mais les valeurs de \(Y\) sont diffÃ©rentes.

  • Liaison T2 (T_2) : Un couple \((i, j)\) est une liaison \(T_2\) si \(X_i \neq X_j\) et \(Y_i = Y_j\). Cela signifie que les valeurs de \(Y\) sont Ã©gales, mais les valeurs de \(X\) sont diffÃ©rentes.

  • Spare : Un couple \((i, j)\) est un spare si \(X_i = X_j\) et \(Y_i = Y_j\). Cela signifie que les valeurs des deux sÃ©ries sont Ã©gales pour les deux observations.

Le problÃ¨me de cet algorithme naÃ¯f est quâ€™il a une complexitÃ© temporelle en \(O(n^2)\), car il y a \(\frac{n(n-1)}{2}\) comparaisons Ã  effectuer.

Un algorithme amÃ©liorÃ© pour le calcul du Tau de Kendall est lâ€™algorithme NDTau. Cet algorithme repose sur lâ€™utilisation dâ€™un jeu de donnÃ©es sans doublons, que lâ€™on trie ensuite avec un algorithme de tri ayant une complexitÃ© de \(O(n \log n)\). Dans notre cas, nous avons choisi dâ€™utiliser un tri fusion.

Lorsque lâ€™on travaille avec un jeu de donnÃ©es sans doublons, les valeurs de \(T_1\) et \(T_2\) sont Ã©gales Ã  zÃ©ro. En effet, comme il nâ€™y a pas de valeurs Ã©gales, il nâ€™y a pas de liaisons. De plus, Ã©tant donnÃ© que la somme des concordances et discordances \(C + D\) est Ã©gale Ã  \(\frac{n(n-1)}{2}\), on obtient une expression simplifiÃ©e pour Tau :

\[ \tau = 1 - \frac{4D}{n(n-1)} \]

oÃ¹ \(D\) est le nombre de discordances.

# Fonction pour gÃ©nÃ©rer des donnÃ©es synthÃ©tiques

generate_correlated_data <- function(n = 100) {

  # GÃ©nÃ¨re un tau de Kendall alÃ©atoire entre -1 et 1
  tau <- runif(1, -1, 1)
  
  # Approximation de la corrÃ©lation de Pearson Ã  partir du tau de Kendall
  rho <- sin(tau * pi / 2)
  
  # GÃ©nÃ©re deux variables normales standard indÃ©pendantes
  X <- rnorm(n)
  Z <- rnorm(n)
  
  # Construit Y avec la corrÃ©lation rho
  Y <- rho * X + sqrt(1 - rho^2) * Z
  
  return(list(X = X, Y = Y, tau = tau, rho = rho))
}

# Fonction pour calculer la corrÃ©lation de Kendall sur un Ã©chantillon bootstrap

bootstrap_kendall <- function(x, y, n_bootstrap) {
  tau_bootstrap <- numeric(n_bootstrap)
  
  for (i in 1:n_bootstrap) {
    # RÃ©Ã©chantillonnage avec remplacement
    sample_indices <- sample(1:length(x), length(x), replace = TRUE)
    x_sample <- x[sample_indices]
    y_sample <- y[sample_indices]
    
    # Calcul de la corrÃ©lation de Kendall pour le rÃ©Ã©chantillon
    tau_bootstrap[i] <- cor(x_sample, y_sample, method = "kendall")
  }
  
  # Retourne les rÃ©sultats bootstrap (moyenne et intervalle de confiance)
  return(c(mean = mean(tau_bootstrap), 
           lower = quantile(tau_bootstrap, 0.025), 
           upper = quantile(tau_bootstrap, 0.975)))
}

# Calcul du N maximum par mÃ©thode pour un temps limite fixÃ© de 10 secondes
find_max_N_power_test <- function(time_limit_sec, base = 10, exp_min = 6, exp_max = 10) {
  start_time <- Sys.time()

  # Fonctions Ã  tester
  methods <- list(
    "Tau_R_naif" = naive_tau_kendall_R,
    "Tau_R_ND" = NDtau_kendall_R,
    "Tau_cpp_naif" = naive_tau_kendall_cpp,
    "Tau_cpp_ND" = NDtau_kendall_cpp
  )
  
  # Liste pour stocker le N_max pour chaque mÃ©thode
  max_N_for_methods <- numeric(length(methods))
  names(max_N_for_methods) <- names(methods)
  
  # Boucle sur chaque mÃ©thode
  for (method in names(methods)) {
    func <- methods[[method]]
    
    max_N_for_method <- 0  # Variable pour stocker le plus grand N testÃ© sous le temps limite
    
    # Tester les puissances de N
    for (exp in seq(exp_min, exp_max, by = 1)) {
      N_test <- base^exp
      data <- generate_correlated_data(N_test)
      
      # Mesurer le temps d'exÃ©cution
      exec_time_sec <- microbenchmark(func(data$X, data$Y), times = 1)$time / 1e9  # Convertir en secondes
      
      # Ajouter le rÃ©sultat si le temps est sous le temps limite
      if (exec_time_sec > time_limit_sec) {
        break  # ArrÃªter dÃ¨s que le temps dÃ©passe
      }
      
      # Mettre Ã  jour le plus grand N pour cette mÃ©thode
      max_N_for_method <- N_test
    }
    
    # Stocker le N_max pour la mÃ©thode
    max_N_for_methods[method] <- max_N_for_method
  }
  
  # CrÃ©er un data frame avec les rÃ©sultats sous forme de ligne
  N_results <- as.data.frame(t(max_N_for_methods))
  colnames(N_results) <- names(methods)
  
  end_time <- Sys.time()  # Fin du chronomÃ¨tre
  total_exec_time_min <- as.numeric(difftime(end_time, start_time, units = "mins"))  # Temps total en minutes
  cat("\nTemps total d'exÃ©cution de la fonction:", round(total_exec_time_min,0), "minutes\n")
  
  return(N_results)
}
